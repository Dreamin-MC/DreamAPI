## How to read a service page

Each service page follows the same structure:

1. **Purpose**: what problem the service solves
2. **Source links**: API interface and core implementation
3. **How it works**: runtime behavior and limitations
4. **Availability**: load mode and registration notes
5. **Examples**: practical snippets you can adapt
